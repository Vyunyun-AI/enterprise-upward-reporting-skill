# Evaluation Cases

## Case 1: Short draft, limited detail
Input: a 2–3 sentence rough note about delayed KPI reporting.
Expected:
- Produces a structured formal report
- Does not invent KPI values
- Includes optimization note

## Case 2: Rectification response
Input: a draft replying to audit findings.
Expected:
- Uses “attitude → facts → causes → rectification closure” structure
- Acknowledges the issue instead of only highlighting positives

## Case 3: Inspection rectification bulletin
Input: multiple issues and actions.
Expected:
- Uses “问题 → 整改措施 → 整改成效” per item
- Ends with long-term mechanism language

## Acceptance checklist
- No fabricated facts or numbers
- Clear formal tone
- Strong compliance closure
- Fits one standard structure model
